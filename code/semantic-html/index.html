Semantic HTML

<header> - introductory content or a set of navigational links

<nav> - defines a set of navigation major links

<main> - the main content of the document

<article> - independent, self-contained content

<section> - a section in a document

<aside> - content aside from the content it is placed in

<section> - a section in a document

<footer> - footer for a document or section

<address> - contact information for the author/owner of a document or an article

Tips:

  • If you don't know what semantic HTML container element you should use, then a <section> is always a safe bet. And it is still okay to use <div> tags for layout; even this page does!
  • Semantic HTML also means using other elements for their purpose, like select, input, fieldset, button, etc.
